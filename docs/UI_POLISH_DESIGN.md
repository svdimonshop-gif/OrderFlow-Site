# OrderFlow-Site — утверждённая UI-доработка

## Цель

Исправить мобильную и desktop-сетку публичного сайта без замены Next.js, GSAP,
GitHub Pages или APK release-пайплайна. Визуальный язык остаётся строгим
операционным интерфейсом: понятные границы, спокойные поверхности и заметная,
но функциональная motion-система.

## Контекст и ограничения

- Публичный сайт: `svdimonshop-gif/OrderFlow-Site`.
- Сайт остаётся static export Next.js и читает APK из GitHub Releases.
- Скрипты в `OrderFlow` не меняются:
  `build_and_publish_release.ps1`, `publish_release_apk.ps1` и их `-WhatIf`.
- Не добавлять зависимости: использовать установленный GSAP, CSS и inline SVG.
- RU остаётся языком по умолчанию; UA и light/dark темы сохраняются.
- `prefers-reduced-motion` отключает все необязательные эффекты.

## Принятые решения

| Решение | Альтернативы | Причина |
| --- | --- | --- |
| Единая мобильная сетка 20 px | Разные 12/16/20 px контейнеры | Шапка, панели и CTA совпадают по краям. |
| Один экран галереи за раз | Показывать край следующей карточки | Соседняя карточка выглядит как обрезание и создаёт светлые «уголки». |
| Метаданные экрана под галереей | Текст и стрелки над экраном | Сохраняет естественный порядок: экран, контекст, управление. |
| Три парные status-карточки | Единая таблица или шесть плиток | Сохраняет смысловые связи без табличного вида. |
| Пятый шаг установки — APK | Отдельная внешняя download-кнопка | Все действия быстрого старта получают единый визуальный вес. |
| SVG-сборщик + CSS/GSAP | Новая анимационная библиотека или bitmap | Нет новых зависимостей, быстрее на Android и проще поддерживать. |

## Сетка и типографика

- Mobile (320–430 px): единый `20 px` боковой отступ для header, content,
  CTA, FAQ, галереи и карточек.
- Desktop: центральный контейнер до `1280 px`; все вложенные поверхности
  используют ширину секции, без дополнительного сужения.
- Шкала: helper — минимум `16 px`, основной текст — `17–18 px`, заголовок
  карточки — `22–24 px`, mobile section heading — `30–36 px`, desktop hero —
  максимум `48 px`.
- Все interactive-контролы имеют высоту не менее `48 px`.

## Рабочий контур

Заменить единую таблицу из шести ячеек тремя отдельными карточками:

1. `1 Маршрут роботи` / `18 Екранів застосунку`.
2. `3 Мови інтерфейсу` / `2 Режими сканера`.
3. `Offline Готовність до слабкої мережі` / `QR Фінальна видача`.

Внутри карточки два показателя отделяются тонкой линией. Акцент получают число
или статус; зелёные подчёркивания удаляются. Desktop показывает три карточки в
ряд, tablet — две плюс одну, mobile — одну карточку в строку.

## 18 экранов и lightbox

- На mobile экран выводится как одна полная карточка общей ширины, без
  отрицательных внешних отступов и без видимого фрагмента соседнего кадра.
- `PhoneFrame` сохраняет intrinsic `width`/`height` и масштабируется через
  `contain`, чтобы исключить CLS и обрезание.
- Блок `счётчик + название + описание + стрелки` переносится под изображение,
  отделяется от него `16 px` и имеет ту же ширину.
- Сохраняются native swipe, drag, scroll-snap, стрелки, клавиатура и lightbox.
- Lightbox получает безопасный нижний отступ и выводит изображение только через
  `contain`.

## FAQ, header и быстрый старт

- FAQ убирает двойной верхний отступ: breadcrumb появляется после safe zone
  фиксированной шапки.
- «Відкрити всю довідку» занимает всю ширину content-grid на mobile.
- Header скрывается при scroll down и возвращается при scroll up; якоря получают
  `scroll-margin-top`.
- Быстрый старт содержит пять идентичных карточек. Последняя — download-card с
  иконкой, версией и прямой ссылкой на `OrderFlow.apk`.

## Motion

- Hero получает видимого inline SVG-сборщика с сумкой и сканером: шаг, движение
  руки, короткий скан-луч и анимированная маршрутная линия.
- Экраны телефона меняются направленной маской `320–420 ms`.
- Reveal запускается при фактическом входе секции в viewport, а не только в
  момент начальной загрузки.
- Status-карточки раскрываются коротким stagger; без bounce, random particles и
  постоянного glow.
- В reduced-motion остаются статичные состояния и мгновенные переключения.

## Визуальная стабильность и проверка

- Убрать полупрозрачность и соседние карточки там, где они создают белые
  «уголки» light theme; поверхности галереи и workflow сделать непрозрачными.
- Проверить 320, 360, 375, 390, 412, 430 px; landscape 640x360 и 915x412;
  desktop 768, 1024, 1280, 1440 и 1600 px.
- На каждой ширине: нет body horizontal scroll, header не закрывает контент,
  CTA совпадают с сеткой, работает свайп/lightbox, APK загружается напрямую,
  Release metadata корректно отображается.
